JSON y su Uso

JSON (JavaScript Object Notation) es un formato de texto ligero para representar datos estructurados. Es fácil de leer y escribir para humanos y fácil de parsear por máquinas. Aunque nació en JavaScript, hoy es universal.

Características Clave

  • Estructura basada en objetos (clave: valor) y arrays.
  • Tipos simples: cadenas, números, booleanos, null, objetos y arrays.
  • Sintaxis concisa y sin etiquetas de cierre como XML.
  • Ideal en APIs REST y comunicación cliente-servidor.

Usos Principales

  • APIs REST / comunicación entre frontend y backend.
  • Archivos de configuración (.json).
  • Bases de datos NoSQL como MongoDB.
  • Mensajería y serialización.
  • Intercambio de datos en móviles y microservicios.

Ventajas / Desventajas

Ventajas: simple, compacto, rápido de parsear, ampliamente soportado.
Desventajas: no tiene un sistema de esquemas tan formal como XSD; menos expresivo que XML para documentos muy complejos.

Ejemplo Sencillo

{
  "note": {
    "to": "María",
    "from": "Juan",
    "heading": {
      "type": "reminder",
      "text": "Recordatorio"
    },
    "body": "Compra pan y leche."
  }
}

Explicación Rápida

  • note es un objeto que contiene to, from, heading y body.
  • heading es otro objeto con un atributo type y el texto en text.
  • JSON usa llaves {} para objetos y corchetes [] para arrays.

Cómo Leer / Parsear

JavaScript

const jsonString = `{
  "note": {
    "to": "María",
    "from": "Juan",
    "heading": { "type": "reminder", "text": "Recordatorio" },
    "body": "Compra pan y leche."
  }
}`;

const data = JSON.parse(jsonString);
console.log(data.note.to); // "María"

Python

import json
json_string = '{"note": {"to":"María","from":"Juan","heading":{"type":"reminder","text":"Recordatorio"},"body":"Compra pan y leche."}}'
data = json.loads(json_string)
print(data['note']['from'])  # Juan

Recomendaciones de Uso

  • Usar JSON para el intercambio de datos entre apps web o móviles.
  • Usar claves claras y descriptivas.
  • Incluir solo la información necesaria.
  • Validar formato (ejemplo: JSONLint).
  • Asegurarse de usar UTF-8 y evitar comentarios.
